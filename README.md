# markdown-bot

Telegram-бот, который принимает markdown (текстом или файлом `.md`/`.txt`)
и отправляет его обратно как rich message — новый формат структурированных
сообщений из Bot API 10.1 (`sendRichMessage`).

## Запуск

1. Получите токен у [@BotFather](https://t.me/BotFather).
2. Установите зависимость: `pip install requests`
3. Запустите:

```bash
BOT_TOKEN=123456:ABC-your-token python3 bot.py
```

## Что умеет

- Текстовое сообщение с markdown → rich message.
- Файл `.md`, `.markdown`, `.txt` (UTF-8, до 256 КБ) → rich message.
- Поддерживается GitHub Flavored Markdown: заголовки, цитаты, ссылки,
  формулы (`$x^2$`, `$$E=mc^2$$`), списки, таблицы, сноски, спойлеры и др.
- Ошибки парсинга и лимиты (32768 символов, 500 блоков) сообщаются в ответ.
- Публикация в канал: `/post @канал`, затем прислать текст или файл.
  `/post` без аргумента использует последний канал, `/cancel` отменяет.
  Бот должен быть администратором канала с правом публикации сообщений.

Rich-формат полноценно отображается в клиентах Telegram Desktop 6.9+
и соответствующих мобильных версиях.

## Веб-редактор (прототип будущей Mini App)

```bash
python3 server.py
```

Токен читается из `.env` (`BOT_TOKEN=...`). Сервер печатает адрес вида
`http://192.168.x.x:8080` — откройте его с телефона в той же Wi-Fi-сети.

- Панель форматирования: заголовки, стили, цитаты, списки, таблицы,
  формулы, сворачиваемые блоки.
- Кнопка «📷 Картинка» загружает файл в S3 (настройки в `.env`:
  `S3_ENDPOINT`, `S3_REGION`, `S3_BUCKET`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`,
  `S3_PUBLIC_BASE` — публичный домен бакета вида `*.selstorage.ru` из
  панели Selectel; бакет должен быть публичным на чтение).
- Кнопка «🔗 Картинка по ссылке» вставляет уже готовый URL.
- Важно: серверы Telegram не могут скачивать медиа напрямую из сетей
  Selectel, поэтому при публикации картинки из S3 (и из хранилища
  Telegram, `tg-file://…`) автоматически перезаливаются на одноразовый
  хостинг litterbox.catbox.moe (живёт 1 час). Telegram скачивает и
  перехостит медиа у себя в момент отправки, так что в доставленном
  посте исходные ссылки не фигурируют и их срок жизни не важен.
- «Превью в Telegram» отправляет пост в ваш чат с ботом (нужно один раз
  написать боту `/start`, чтобы он запомнил ваш chat_id в `state.json`).
- «Опубликовать» отправляет пост в указанный канал.
- Черновик сохраняется в браузере и переживает перезагрузку страницы.

Сервер рассчитан на локальную сеть — не публикуйте порт наружу, авторизации нет.
