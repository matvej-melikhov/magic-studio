# Студия постов (magic-studio)

Сервис для оформления постов Telegram-каналов в rich-формате (Bot API 10.1,
`sendRichMessage`): Telegram-бот + веб-редактор с точным предпросмотром,
черновиками, отложенной публикацией и AI-помощником.

## Структура репозитория

```
app/        Python-код
  bot.py         Telegram-бот: markdown → rich message, /post, /login
  server.py      веб-сервер редактора: API, публикация, планировщик, AI-прокси
  storage.py     SQLite-хранилище (studio.db)
  s3_utils.py    загрузка медиа в S3 (ручная подпись SigV4, без boto3)
  env_utils.py   чтение .env
web/        Фронтенд
  editor.html    весь веб-редактор (один файл)
  manifest.json  PWA-манифест + иконки приложения
deploy/     systemd-юниты и скрипты развёртывания (prod и dev)
.github/    CI/CD: деплой dev по push в ветку dev, prod — по тегу v*
```

Рабочие файлы (`.env`, `studio.db*`, `state.json`, `login_codes.json`)
живут в корне, в git не попадают и деплоем не перезаписываются.

## Локальный запуск

```bash
pip install -r requirements.txt
cp .env.example .env   # или создайте .env: TELEGRAM_KEY=<токен от @BotFather>
python3 app/bot.py     # бот (в отдельном терминале)
python3 app/server.py  # веб-редактор на http://localhost:8080
```

Вход в редактор — через Telegram (deep-link на бота) или по коду `/login`.

## Возможности

- Полный rich-markdown: заголовки, стили, спойлеры, формулы LaTeX, таблицы,
  сноски, сворачиваемые блоки, коллажи/слайдшоу, карты, дата-время.
- Предпросмотр, повторяющий рендер iOS-клиента Telegram.
- Медиа: загрузка в свой S3 (Selectel) или хранилище Telegram; при публикации
  файлы перезаливаются через хостинг-трамплин — Telegram перехостит их у себя,
  исходные ссылки в посте не фигурируют.
- Каналы (с проверкой прав администратора), черновики, отложенные посты
  (серверный планировщик), редактирование опубликованного.
- AI-помощник (Ollama, локальная или облачная модель): рерайт, авторазметка,
  генерация с нуля — со стримингом токенов прямо в редактор.

## Стенды и деплой

| Стенд | Где | Деплой |
|---|---|---|
| local | мак разработчика | руками |
| dev   | сервер, порт 8081 | push/merge в ветку `dev` |
| prod  | сервер, порт 80   | тег `v*` на `main` |

У каждого стенда свой бот (токен в `.env` стенда) и своя база — поэтому
экземпляры не конфликтуют за getUpdates. Деплой: GitHub Actions →
rsync → `deploy/update.sh <стенд>` (секреты `DEPLOY_HOST`, `DEPLOY_USER`,
`DEPLOY_SSH_KEY`). Первичная настройка сервера — `deploy/setup.sh`.
